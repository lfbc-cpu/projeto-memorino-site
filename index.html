Descrição do Projeto

Motivação, Objetivo e Contextualização

Motivação: A escolha de desenvolver um jogo de memória é motivada pela combinação da aprendizagem de eletrônica/programação com os benefícios cognitivos, como estímulo à atenção e memória de curto prazo.

Objetivo: Desenvolver um protótipo funcional usando Arduino UNO, botões e display LCD I2C que exiba fases e pontuação, aumentando a dificuldade da sequência a cada nível.

Contextualização: Aplicável em ambientes educacionais e feiras de ciências, demonstrando conceitos de interfaces físicas, timers e aleatoriedade controlada.

Componentes e Suprimentos

  • Arduino UNO R3: Placa microcontroladora principal.
  • Display LCD 16x2 com I²C: Exibição de dados usando interface PCF8574 para reduzir fios.
  • Botões táteis (3x): Entrada de dados do usuário.
  • LEDs (4x): Indicadores visuais da sequência.
  • Resistores 220Ω (3x): Proteção para os LEDs.
  • Buzzer Piezo Passivo: Feedback sonoro (notas musicais).
  • Protoboard e Jumpers: Para conexões e montagem sem solda.

Aplicativos e Plataforma

Foram utilizados no desenvolvimento:

  • Arduino IDE (v2.3.6)
  • Bibliotecas: LiquidCrystal_I2C e Wire
  • Git e GitHub para versionamento

Passo a passo da Montagem

Passo inicial: reúna os componentes necessários

Reúna o Arduino, LCD I2C, 3 botões, 3 LEDs, buzzer, resistores, jumpers e protoboard.

Passo 1: Conectar o LCD I2C

Conecte VCC ao 5V, GND ao GND do arduino, SDA ao pino A4 e SCL ao pino A5 do Arduino UNO.

Passo 2: Montagem dos Botões e dos LEDs

Posicione os 3 botões na protoboard juntamente com os LEDs da respectiva cor, conforme imagem. Conecte cada um ao GND e a um pino digital (D5, D6, D7).

Passo 3: Conexão dos LEDs

Conecte os LEDs (ânodo no pino por meio de um resistor e cátodo no GND) e os botões (uma perna no pino D5 e a outra no GND). Lembre-se de que o botão ligado ao pino D5 deve ser posicionado ao lado do LED do D2, ambos com a mesma cor. Da mesma forma, o botão do D6 deve ficar ao lado do LED do D3, também com cores correspondentes. Por fim, o botão do D7 deve ser colocado ao lado do LED do D4, mantendo a mesma cor entre eles.

Passo 4: Conectar o Buzzer

Conecte o terminal positivo do buzzer ao pino 8 e o negativo ao GND.

Passo 5: Verificar conexões

Inspecione tudo em busca de curto-circuitos e ligue o Arduino via cabo USB para garantir que o LED "ON" acenda.

Passo 6: Upload do Código

Abra o Arduino IDE, se necessário instale a biblioteca do LiquidCrystal I2C, carregue o sketch e envie para a placa. Verifique se não há erros de compilação.

Passo 7: Ajustes Finais

Teste as sequências. Se estiver muito rápido, ajuste o delay no código. Verifique o contraste do LCD.


Passo 8: Projeto Concluído

O Memorino está pronto para ser jogado!

Código Fonte

O código foi desenvolvido em C++ utilizando a IDE do Arduino. Abaixo, destacamos as partes mais importantes da lógica do jogo.

1. Configuração e Variáveis

Neste trecho, importamos a biblioteca do LCD via I2C e definimos os pinos dos LEDs e Botões utilizando arrays (vetores), o que facilita a manipulação dentro dos loops.


#include <Wire.h>
#include <LiquidCrystal_I2C.h>

// Configuração do Display e Pinos
LiquidCrystal_I2C lcd(0x27, 16, 2);
const int leds[] = {2, 3, 4};
const int botoes[] = {5, 6, 7};
const int buzzer = 8;

// Variáveis de Estado
const int MAX_FASES = 10;
int sequencia[MAX_FASES];
int tamanhoSequencia = 0;

2. Lógica de Verificação (Core do Jogo)

A função verificarJogada é o coração do Memorino. Ela compara o botão pressionado pelo jogador com a sequência armazenada. Se o jogador acertar toda a sequência da fase atual, o jogo avança; caso contrário, o jogo reinicia.


void verificarJogada(int botaoPressionado) {
  // Verifica se o botão pressionado corresponde à sequência correta
  if (botaoPressionado == sequencia[indiceJogador]) {
    tone(buzzer, 1000, 200);
    indiceJogador++;

    // Se completou a sequência da fase
    if (indiceJogador == tamanhoSequencia) {
      // Verifica vitória (Fase 10) ou avança fase
      if (tamanhoSequencia == MAX_FASES) {
        lcd.print("VOCE VENCEU!");
        iniciarJogo();
      } else {
        tamanhoSequencia++;
        sequencia[tamanhoSequencia - 1] = random(0, 3);
        jogando = false; // Prepara para mostrar nova sequência
      }
    }
  } else {
    // Game Over em caso de erro
    tone(buzzer, 200, 1000);
    lcd.print("Voce perdeu!");
    iniciarJogo();
  }
}

Problemas e Lições Aprendidas

Desafios Enfrentados

  • Configuração do LCD I2C: Inicialmente, o display ligava mas não mostrava os caracteres. O problema foi resolvido ajustando o potenciômetro de contraste (na parte de trás do módulo) e confirmando o endereço hexadecimal 0x27 no código.
  • Leitura dos Botões (Bounce): O Arduino lia múltiplos toques quando apertávamos o botão apenas uma vez. Resolvemos isso implementando um pequeno delay no código para estabilizar o sinal (técnica de Debounce).
  • Mal contato na Protoboard: Durante a montagem, alguns LEDs falharam. A solução foi revisar todas as conexões e padronizar os jumpers (fios), garantindo que o circuito estivesse firme.

Lições Aprendidas

  • Integração Hardware-Software: Aprendemos como a lógica de programação controla componentes físicos e a importância da temporização correta.
  • Protocolo I2C: Compreendemos como esse protocolo facilita o projeto, economizando portas do Arduino ao usar apenas dois pinos para o display.
  • Trabalho em Equipe e GitHub: A experiência reforçou a importância de documentar o processo e usar controle de versão para gerenciar as alterações no código.
